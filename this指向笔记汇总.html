JavaScript中的this

在JavaScript中，this是动态绑定的，或者也可以称为运行时绑定的，这就导致JavaScript中this关键字有能力具备多重含义，带来灵活性的同时，也给初学者带来不少的困惑。因此 本节课 就这一问题进行讨论。

函数中的this

在JavaScript中，this关键字只能在函数内部使用，并且是只读的。因此讨论 JavaScript 中的this，本质就是探讨 函数中的this问题。

下面图形是展示 JavaScript中函数的执行的大致流程：

由于在JavaScript中 this 是动态绑定的，因此 只有在函数执行时，才会去绑定this的值。

其实就是说，在JavaScript中this的值 完全取决于 函数的调用方式。

而在JavaScript中，函数的调用方式有以下几种：作为对象方法调用、作为函数调用、作为构造函数调用、以及使用call/apply调用。

作为对象方法调用

在JavaScript中，函数也是一种数据类型，因此可以作为一个对象的属性，此时函数被称为该对象的方法，在使用这种调用方式时，this自然被被绑定到该对象。

举个栗子：

var obj = {
  name: 'obj',
  fn: function(){
    console.log( this );      // this -> obj对象
    console.log( this.name ); // 这里就会打印出'obj'
  }
};

作为函数调用

函数也可以直接被调用，此时this默认绑定到全局对象。在浏览器中，window就是全局对象。

在下面的栗子中this就是window对象

function foo(){
  console.log( this );
}

作为构造函数调用

JavaScript支持面向对象式编程，与主流面向对象编程语言不同，JavaScript并没有类的概念，而是使用基于原型的继承方式。相应的，JavaScript中的构造函数也很特殊，如果不使用new调用，则和普通函数一样。作为约定俗成的规定，构造函数的名字以大写字母开头，提醒调用者使用正确的方式调用。

作为构造函数调用时，this绑定到新创建的对象上。 看栗子

function Point( x, y ){
  this.x = x;
  this.y = y;
  console.log( this );
}

使用call/apply调用

在JavaScript中，函数也是对象，对象则有属性和方法。call和apply就是函数对象的方法。这两个方法能够改变this所绑定的对象。

看下面的栗子

function fn(){
  console.log( this );
}
fn(); // 这里 this 就是 window对象
var obj = { name: 'obj' };
fn.call( obj ); // 这里 this 就是 obj对象
fn.apply( obj ); // 这里 this 也是 obj对象
